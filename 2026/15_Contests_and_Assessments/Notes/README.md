# Topic Notes: Structured Documentation

Use this folder to document your conceptual understanding. Do not just copy-paste; explain the logic in your own words.

## Documentation Rules
1. Template Usage: Use the provided `TEMPLATE.md` for every new concept.
2. Visuals: Describe the state transitions or draw diagrams (using ASCII or Mermaid).
3. No Clutter: High-level logic goes here. Implementation details go in the Code folder.
