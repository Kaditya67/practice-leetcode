## 🗺️ How to Use This Repository (The Daily Workflow)

### Step 1: Pick a Topic
Check the `MASTER_PLAN.md` to see where you left off. The folders are numbered `01` to `15` in the exact order you should learn them.
*Rule: Do not skip folders.*

### Step 2: Learn the Theory
Navigate into your chosen topic folder (e.g., `01_Foundations_and_STL`). Do not code yet.
1. Open the **`STUDY_GUIDE.md`** inside that folder. Read the logic, triggers, and prerequisites.
2. If you need to write notes, duplicate the `TEMPLATE.md` inside the `Notes/` subfolder and explain it to yourself.

### Step 3: Write the Code
Once you understand the logic, solve the problems listed in your `MASTER_PLAN.md`.
1. Go to the `Code/` subfolder.
2. View the `Code/README.md` to see the strict naming convention.
3. Save your `.cpp` file.

### Step 4: Sync Your Progress
Update manually in 5 minutes using `SYNC_PROGRESS.md`.
1. Add one session row in `TRACKER.md`.
2. Check completed atomic items in `MASTER_PLAN.md`.
3. Log difficult problems in `SPACED_REPETITION_LOG.md`.
4. If needed, ask me to draft exact entries before you paste them.
