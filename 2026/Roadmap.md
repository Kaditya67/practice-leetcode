# 2026 DSA Acceleration Roadmap

This roadmap is aligned to numbered folders (`01` to `15`) and a LeetCode-first strategy with dual goals:
interview quality plus contest speed.

---

## Core Defaults
- Daily focus: 90 minutes.
- Platform mix: 80% LeetCode, 20% GFG/CodeChef.
- Language: C++.
- Tracking: Manual Markdown only.
- Progression style: Accuracy-first.
- March timing contract:
  - Medium median <=35 minutes first, then <=30 minutes.
  - Hard timing target deferred until Medium stability is achieved.

## 90-Minute Session Blueprint
1. 15 minutes: Spaced repetition review (1 old problem).
2. 50 minutes: New learning (1 Medium or 2 Easy if stuck >35 min).
3. 20 minutes: Speed rerun (resolve a past solved problem without notes).
4. 5 minutes: Update tracker and daily planner.

## Weekly Cadence
- Monday to Friday: Standard 90-minute session.
- Saturday: 90-minute timed contest simulation (2 to 3 problems).
- Sunday:
  - 45 minutes weak-area repair.
  - 45 minutes weekly review and next-week topic lock.
  - 10 minutes content sync check across `README.md`, `Roadmap.md`, `MASTER_PLAN.md`, `CURRICULUM.md`.

---

## Folder-Aligned Phases

### Phase 1 (Weeks 1 to 2): Foundations
- Folders: `01_Foundations_and_STL`, `02_Mathematics`.
- Interview track:
  - Complexity, STL behavior, modular arithmetic, sieve.
  - Target: clean implementation and complexity proof.
- Contest-speed track:
  - Fast template setup, overflow-safe arithmetic, quick prime checks.
  - Target: reduce setup time and avoid avoidable WA/TLE.

### Phase 2 (Weeks 3 to 5): Linear Patterns
- Folders: `03_Arrays_and_Searching`, `04_Sorting_and_Greedy`, `05_Strings`.
- Interview track:
  - Sliding window, two pointers, binary search on answer, greedy exchange arguments.
  - Target: pattern recognition with edge-case coverage.
- Contest-speed track:
  - Rapid pointer debugging, off-by-one defense, I/O efficient coding style.
  - Target: reliable first accepted solution under time pressure.

### Phase 3 (Weeks 6 to 8): Recursive and Structural Thinking
- Folders: `06_Recursion_and_Backtracking`, `07_Linked_Lists`, `08_Stacks_and_Queues`, `09_Trees_and_BST`.
- Interview track:
  - Backtracking pruning, monotonic stack/queue, tree traversals and BST invariants.
  - Target: explain recursion state and stack behavior clearly.
- Contest-speed track:
  - Iterative alternatives, stack templates, tree dry-run speed.
  - Target: convert ideas to code quickly with fewer bugs.

### Phase 4 (Weeks 9 to 11): Priority and Optimization
- Folders: `10_Heaps_and_Priority_Queues`, `11_Dynamic_Programming`.
- Interview track:
  - Heap patterns and DP state design (1D/2D/knapsack/tree DP).
  - Target: derive recurrence and justify transitions.
- Contest-speed track:
  - Time-budget DP selection, memory compression, heap tuning.
  - Target: avoid overcomplicated DP during contests.

### Phase 5 (Weeks 12 to 14): Connectivity and Advanced Structures
- Folders: `12_Graphs`, `13_Tries_and_Segment_Trees`, `14_Advanced_Topics`.
- Interview track:
  - BFS/DFS/shortest path/MST, tries, segment trees, DSU basics.
  - Target: pick the right graph/data-structure model quickly.
- Contest-speed track:
  - Reusable graph scaffolds, log-time query structures, stress testing.
  - Target: solve mixed constraint problems with confidence.

### Phase 6 (Ongoing): Assessments
- Folder: `15_Contests_and_Assessments`.
- Interview track:
  - Weekly mixed interview set with post-analysis.
- Contest-speed track:
  - Timed virtual contests and error taxonomy review.
